# Vibber - AI Agent Cloning Platform
## Product Strategy & Market Analysis

**Version:** 1.0
**Date:** January 2026
**Price Point:** $20/employee/month

---

## Executive Summary

Vibber enables employees to create AI agent clones of themselves that can autonomously handle routine tasks across multiple enterprise platforms (Slack, GitHub, Jira, Confluence, Elastic). The platform provides a dashboard for monitoring agent interactions and handling escalations.

## 2. UX Design Philosophy (UX Designer Perspective)

### 2.1 Core UX Principles

1. **Progressive Disclosure** - Start simple, reveal complexity as needed
2. **Trust Through Transparency** - Always show what the agent is doing
3. **Human-in-the-Loop** - Easy escalation, approval workflows
4. **Minimal Friction Onboarding** - Clone creation in under 5 minutes
5. **Confidence Indicators** - Visual cues for agent certainty levels

|------------|----------------|

- `channels:history`, `channels:read`

## 6. Database Schema

```sql
-- Core Tables
CREATE TABLE organizations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    slug VARCHAR(100) UNIQUE NOT NULL,
    plan VARCHAR(50) DEFAULT 'starter',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id UUID REFERENCES organizations(id),
    email VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(255) NOT NULL,
    avatar_url TEXT,
    role VARCHAR(50) DEFAULT 'member',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE agents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    name VARCHAR(255) NOT NULL,
    status VARCHAR(50) DEFAULT 'training',
    confidence_threshold INTEGER DEFAULT 70,
    auto_mode BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE integrations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    agent_id UUID REFERENCES agents(id),
    provider VARCHAR(50) NOT NULL,
    access_token TEXT ENCRYPTED,
    refresh_token TEXT ENCRYPTED,
    scopes TEXT[],
    status VARCHAR(50) DEFAULT 'active',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    expires_at TIMESTAMPTZ
);

CREATE TABLE interactions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    agent_id UUID REFERENCES agents(id),
    provider VARCHAR(50) NOT NULL,
    interaction_type VARCHAR(50) NOT NULL,
    input_data JSONB,
    output_data JSONB,
    confidence_score INTEGER,
    status VARCHAR(50) DEFAULT 'completed',
    escalated BOOLEAN DEFAULT FALSE,
    human_feedback VARCHAR(50),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE training_samples (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    agent_id UUID REFERENCES agents(id),
    provider VARCHAR(50),
    sample_type VARCHAR(50),
    input_text TEXT,
    output_text TEXT,
    embedding VECTOR(1536),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE escalations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    interaction_id UUID REFERENCES interactions(id),
    agent_id UUID REFERENCES agents(id),
    reason TEXT,
    priority VARCHAR(50) DEFAULT 'medium',
    status VARCHAR(50) DEFAULT 'pending',
    resolved_by UUID REFERENCES users(id),
    resolved_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_interactions_agent_id ON interactions(agent_id);
CREATE INDEX idx_interactions_created_at ON interactions(created_at);
CREATE INDEX idx_escalations_status ON escalations(status);
CREATE INDEX idx_training_samples_embedding ON training_samples USING ivfflat (embedding vector_cosine_ops);
```
